## User Progress Page (/sheets/[slug]/progress/[username]) – ASCII Wireframe

all component should be create under parts folder


This page shows a specific user’s progress on a sheet. It includes:
- User avatar, username, sheet name.
- Progress summary (solved, revision completed, combined percentage).
- Pie chart (combined solved + revision progress).
- Target date, joined date, completion status.
- Shareable link (copy button).
- List of all questions in the sheet (river‑style) showing the user’s status for each.

The design is consistent with the sheet details page and uses the same `GoalQuestionItem` component.

### 4. Chart Data Mapping (same as sheet details)

Uses endpoint: `/api/v1/sheets/:slug/progress/:username/chart`

```json
{
  "solved": { "count": 0, "remaining": 2, "percentage": 0 },
  "revision": { "completed": 0, "remaining": 2, "percentage": 0 },
  "totalQuestions": 2
}
```

**Combined percentage** = `(solved.count + revision.completed) / (totalQuestions * 2) * 100`.  
Pie chart displays this percentage as a filled segment.

---

### 5. Copy Link Button

- Clicking “Copy Link” copies the current page URL (`/sheets/[slug]/progress/[username]`).
- Shows a toast: “Link copied to clipboard”.

---

### 6. Edge Cases

- If user has not joined the sheet → 404 / “User not found or not a member” (handled by backend).
- If no questions in sheet → show empty state.
- If all questions completed → show “Fully completed! 🎉” badge.
