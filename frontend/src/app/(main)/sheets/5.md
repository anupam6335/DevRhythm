## Create / Import Sheet (/sheets/create) – ASCII Wireframe

all component should be create under parts folder


The sheet creation flow supports two methods: **Manual** (enter questions one by one) and **Import** (upload Excel/CSV/JSON). The UI uses tabs to switch between the two. The design is responsive, with clear error handling and preview for imports.

### 5. Handling Unresolved Identifiers (Import)

When the backend returns `data.unresolved` (array of strings), the UI shows a list of unresolved items. Each unresolved row can be:
- Manually mapped to a correct question title/slug via a small inline edit or a modal.
- Skipped (the row will be omitted from sheet creation).
- The user can also “Proceed anyway” which creates the sheet only with successfully resolved questions.

**Example unresolved modal (desktop):**

```
┌─────────────────────────────────────────────────────┐
│  ⚠️ Unresolved Questions                    [Close] │
```

### 6. Validation & Error States

- **Manual tab:** Name required, target date required (future), at least one question selected.
- **Import tab:** File required, at least one valid question after resolution.
- **Duplicate sheet name:** backend returns 409. Show error message: “Sheet with this name already exists. [View existing sheet]”.
- **Network errors:** toast notification with retry button.
- **Loading states:** spinner on Create button, file upload progress indicator.
