## Sheet Details Page (/sheets/[slug]) – ASCII Wireframe (with Pie Chart)

all component should be create under parts folder


The page includes:
- Hero section (sheet metadata, join/leave, edit/delete)
- **If user has joined:** Pie chart (combined solved + revision progress) and summary stats (solved count, revision completed count)
- **If not joined:** No chart, only call‑to‑action to join
- Participants list (avatars)
- Questions grouped by tags (river‑style, with status and participant solved counts)

The pie chart is a single donut/pie representing **overall completion** – e.g., `(solved.count + revision.completed) / (totalQuestions * 2)`.

### 5. Pie Chart Data Mapping

Using the provided API endpoint `/sheets/:slug/progress/me/chart`:

```json
{
  "solved": { "count": 0, "remaining": 2, "percentage": 0 },
  "revision": { "completed": 0, "remaining": 2, "percentage": 0 },
  "totalQuestions": 2
}
```

**Chart calculation:**
- Total possible completions = `totalQuestions * 2` (each question can be solved AND revision completed).
- Completed count = `solved.count + revision.completed`.
- Percentage = `(completed / (totalQuestions * 2)) * 100`.

In the example: `(0 + 0) / (2 * 2) = 0%`.

The pie chart displays this percentage. Use Chart.js to create a donut/pie chart with two segments: completed (green) and remaining (light grey). The chart is placed in the right column on desktop, top on tablet/mobile.
